# Create Work Page

Create a new work entry in `content/work/`.

Requirements:

- describe the product or project clearly without hype
- keep the framing premium, composed, and operationally credible
- include the standard frontmatter and use optional `video` or `appStoreUrl` only when real
- write for a future public work page, not an internal project note
- structure the body so it can later expand into a richer page without being rewritten from scratch

Output should be one Markdown file that fits the existing repo structure.
